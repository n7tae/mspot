# #################### Reflectors ###########################
#
# This is either for "overrides" where you are providing new, correct data for
# an unknown version string, or fixing a url, or some other parameter.
#
# It might also be a place to define the connection to a reflector that is
# not registered at DVref.com.
#
# 'Special-modules' for M17 reflectors will pass encrypted voice data.
# 'Special-modules' for URF reflectors are fully transcoded.
#
# Reflector;Version;Modules;Special-modules;IPv4-address;IPv6-address;Port;Dashboard-URL


# ################## Direct Routing Targets ##################
#
# You do not link to these target, but you load them the same way you link to a reflector:
# Put the destination in DST and quick-key.
# Capabilities: 2 chars:
# 1. Data handling:
#    'S' if only stream data is processed
#    'P' if only packet data is processed
#    'B' if both packet and stream data is processed
# 2. TYPE handling:
#    'L' if only legacy TYPE format is used
#    '3' if only M17 Specification V#3 TYPE format is used
#    'B' if both formats are understood
# The example shows that N0CALL will only receive Stream data at 44.46.48.201:17100, and uses the legacy TYPE format
# NOCALL:SL;44.46.48.201;;17100
# Destination;Capabilities;IPv4Address;IPv6Address;Port
